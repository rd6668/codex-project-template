# Example Web App

A small example project that shows how this template can be filled in for a long-running software product.

## Overview

- Problem: teams need a simple internal app to manage work items and project visibility
- Users: managers, contributors, and internal operators
- Status: Active Development

## Quick Start

### Requirements

- Node.js 20+
- PostgreSQL 15+

### Setup

```bash
npm install
cp .env.example .env
```

### Run

```bash
npm run dev
```

## Documentation

- `PROJECT.md`: project goals, scope, constraints
- `ARCHITECTURE.md`: system structure and boundaries
- `DECISIONS.md`: major technical decisions
- `TODO.md`: project task list
- `API.md`: API contracts
- `DATABASE.md`: data model and runtime state notes
- `STYLE_GUIDE.md`: engineering conventions
- `WORKFLOW.md`: collaboration and document update rules
- `PROMPTS.md`: fixed prompts for repeated AI collaboration

## Current Scope

- team work item management
- project status visibility
- basic role-based access control

## Notes

- this example is intentionally generic and exists only to demonstrate how the template can be populated
