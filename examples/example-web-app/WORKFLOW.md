# Workflow

## Document Roles

- `README.md`: public entry point and quick start
- `PROJECT.md`: project goals, scope, constraints, non-goals
- `ARCHITECTURE.md`: structure and boundaries
- `DECISIONS.md`: major decisions and rationale
- `TODO.md`: task tracking
- `API.md`: interface contracts
- `DATABASE.md`: persistent data design and runtime state notes
- `STYLE_GUIDE.md`: coding conventions
- `PROMPTS.md`: fixed prompts for repeated AI collaboration

## Update Rules

After any major change, check whether these need updates:

- `ARCHITECTURE.md`
- `DECISIONS.md`
- `API.md`
- `DATABASE.md`
- `TODO.md`

## Definition Of Major Change

- architecture changes
- interface changes
- data model changes
- key dependency changes
- permission model changes

## Delivery Rules

- prefer small validated changes
- document important decisions
- keep docs aligned with the real system
